Resets in OLC

Method 1 Method 2

Resetting mobs, or objects in rooms in OLC isn't all that difficult. If you want to see how to make shops, see the section on shops. This section deals with just loading a mob into a room as a reset, or placing an object in a room/in a room on a mob/in a room in another container.

Method 1

Mobs

The easiest way to reset mobs is to go to the room you want the mob to originate from. Then type "redit".

"mreset (vnum of mob) (global limit) (room limit)"
mreset = mob reset.
vnum of mob is self-explanatory.
Global limit is the total number of mobs in the world.
Room limit is the total number of mobs to be reset in that room.

Objects

Object resets are a little more difficult, because of more possibilities with where and how to put an object. Once again enter "redit".

"oreset (vnum of object)" will reset an object with specified vnum on the ground in that room.

"oreset (vnum of object) inside (name of container in room)" resets that object inside a container already in the room. If you want more than one, just repeat the resets.

"oreset (vnum of object) (keyword of mob) (wear location)" resets an object on the mob that is also reset to the room, on the specified wear location. Type "? wear-loc" to see where you can load the object.

Method 2

Setting mobs outside of redit:

"reset # mob (vnum of mob) (global limit) (room limit)"
where the # is the reset number on the list, when you type "resets" in the room.
Global limit is the total number of mobs in the world/area.
Room limit is the total number of mobs to be reset in that room.

Setting objects outside of redit:

"reset # obj (vnum of object) room"
Sets the object inside the room.

"reset # obj (vnum) inside (vnum) (global limit) (count)"
Sets an object within a container. If no limit or count is set, only one object will be set in the entire world, and the object will only load once per reboot/crash.

"reset # obj (vnum) (wear_loc)"
Sets the equipment to load on the mob that is reset to the room, on the specified wear location. Please note that the object reset comes after the mob, e.g. mob number is 1, object number should be 2. Type "? wear-loc" to see where you can load the object.

General reset commands

"reset # delete"
Deletes the specified reset. 
RESET <number> RANDOM [#x exits]
