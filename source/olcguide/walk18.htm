If the program fails for any reason, change it in pedit, then remove the program from the mobile/object and add it again, even after you purge the existing mob/object and load a new one. This means "delmprog #", e.g. in the current example type "delmprog 0", then typing "addmprog 1000 grall 100". The code is a bit peculiar in this, in that if you change the code, even if you load a new mob/object, the mprog will not update/function.

Finally, if something doesn't work after a couple of tries, ask a builder or an area master. Buggy programs can crash the MUD, which will not be fun.
