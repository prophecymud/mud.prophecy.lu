4)   Writing the description (append mode)

For zMUD users, “.” is the default movement character, so they will either need to change/disable it (by going into Preferences > Special Characters) or disable parsing, in order to use the "dot commands".

To write the room description, type "desc" in redit. You will enter the append mode and see this:

desc

[Room-Edit]
-=======- Entering APPEND Mode -========-
Type .h on a new line for help
Terminate with a ~ or @ on a blank line.
-=======================================-

>

In order to write the room description, I will type it directly into the MUD. So I type "Wooden drawers cover all the walls in this small room, save for where the door is. Small bronze plaques have been nailed onto each drawer, showing the name of the herb found within." and press enter. To see what I've written, I will type ".s", as shown below.

Wooden drawers cover all the walls in this small room, save for where the door is. Small bronze plaques have been nailed onto each drawer, showing the name of the herb found within.
.s

> 1 Wooden drawers cover all the walls in this small room, save for where the door is. Small bronze plaques have been nailed onto each drawer, showing the name of the herb found within.

I will continue typing in the room description, until I am satisfied I have written enough. I type ".s" now and see:

Close to the northern wall, a polished counter sits opposite the door, and on it are a book, a pot of ink and a quill.
.s

> 1 Wooden drawers cover all the walls in this small room, save for where the door is. Small bronze plaques have been nailed onto each drawer, showing the name of the herb found within.
2 Close to the northern wall, a polished counter sits opposite the door, and on it are a book, a pot of ink and a quill.

I then realise that I have forgotten to add a line between line 1 and line 2, since I intended to make the shopkeeper the only person who understands the plaques. So I now add a line by typing ".i 2 However, the names cannot be understood, as they have been written in strange letters." This means I insert a line in front of line 2. I type ".f" and see this:

However, the names cannot understood, as they have been written in strange letters.
.s

> 1 Wooden drawers cover all the walls in this small room, save for where the door is. Small bronze plaques have been nailed onto each drawer, showing the name of the herb found within.
2 However, the names cannot understood, as they have been written in strange letters.
3 Close to the northern wall, a polished counter sits opposite the door, and on it are a book, a pot of ink and a quill.

It is not formatted, and looks very messy. So to make it look nicer, I type ".f". This formats the room description so that the line stops at around 80 characters. I then type ".s" and now I see this:

.f
.s

> 1 Wooden drawers cover all the walls in this small room, save for where the
2 door is. Small bronze plaques have been nailed onto each drawer, showing
3 the name of the herb found within. However, the names cannot understood, as
4 they have been written in strange letters. Close to the northern wall, a
5 polished counter sits opposite the door, and on it are a book, a pot of ink
6 and a quill.

Oops. I made a grammatical error. I should be "the names cannot be understood", but I missed out the "be". To add it, I type ".r "cannot understood" "cannot be understood"". This means I replace the first "cannot understood" in the description with "cannot be understood". I type ".f" and ".s" to make sure there are no more typos.

.r "cannot understood" "cannot be understood"
.f
.s

> 1 Wooden drawers cover all the walls in this small room, save for where the
2 door is. Small bronze plaques have been nailed onto each drawer, showing
3 the name of the herb found within. However, the names cannot understood, be
4 as they have been written in strange letters. Close to the northern wall, a
5 polished counter sits opposite the door, and on it are a book, a pot of ink
6 and a quill.

Very nice. But it looks a bit plain without colour. To add colour to "plaques" for example, I would type ".r "plaques" "`3pl`#aqu`3es``"". I exit the append mode and look at the room. Voila!

Inside a shop
  Wooden drawers cover all the walls in this small room, save for where the
door is. Small bronze plaques have been nailed onto each drawer, showing
the name of the herb found within. However, the names cannot be understood,
as they have been written in strange letters. Close to the northern wall, a
polished counter sits opposite the door, and on it are a book, a pot of ink
and a quill.

Now, it might seem odd that I don't format it, so let's see what happens if I format it with colour:

desc
.f
.s

> 1 Wooden drawers cover all the walls in this small room, save for where the
2 door is. Small 1b3r#on3z1e 3pl#aqu3es have been nailed onto
3 each drawer, showing the name of the herb found within. However, the names
4 cannot be understood, as they have been written in strange letters. Close
5 to the northern wall, a polished counter sits opposite the door, and on it
6 are a book, a pot of ink and a quill.

So far so good. So I exit the append by typing "@", and look at the room again.

@

Inside a shop
  Wooden drawers cover all the walls in this small room, save for where the
door is. Small bronze plaques have been nailed onto
each drawer, showing the name of the herb found within. However, the names
cannot be understood, as they have been written in strange letters. Close
to the northern wall, a polished counter sits opposite the door, and on it
are a book, a pot of ink and a quill.

Bad formatting. The editor reads the colour characters (i.e. ` ) as individual characters too, which is why we do not format room descriptions with colours.
